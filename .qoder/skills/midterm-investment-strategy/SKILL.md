# 中线投资策略 Skill

## 用途

应用一套完整的 A 股中线（3-12 个月）投资决策框架，覆盖从选股到退出的全流程，支持两套平行框架：
- **周期拐点策略**：传统周期股，靠"底部买入→拐点确认→估值修复"赚钱
- **高景气成长股策略**：技术革命/新需求驱动的赛道（如 AI/CPO），靠"渗透率提升+预期差"赚钱

## 激活场景

当用户提出以下类型的请求时启用本 Skill：

- 评估某只股票是否值得建仓（"帮我看看 XX 能不能买"）
- 分析某行业是否处于拐点（"XX 行业是不是反转了"）
- 判断某股票属于周期股还是高景气股
- 复盘当前持仓是否需要调整（"XX 季报出了，要不要继续拿"）
- 决定是否触发止损（"XX 跌了 20%，该不该走"）
- 检验预期差是否成立
- 检查组合层面风险（关联性、集中度）
- 分析外围环境是否适合建仓（流动性/汇率/北向）

## 工作流程

### 第 1 步：识别场景类型

根据用户请求归类到以下场景之一：

| 场景 | 触发关键词 | 加载模块 |
|---|---|---|
| 新建仓评估 | "能不能买"、"值不值得建仓" | 全流程 6 关 + 相关模块 |
| 框架判断 | "周期股还是成长股"、"用哪个框架" | `01`、`02`、`07` |
| 财报分析 | "财报怎么样"、"业绩如何" | `01` |
| 行业拐点 | "行业拐点"、"是不是反转" | `02` 或 `07` |
| 估值检查 | "估值贵不贵"、"PE/PEG 怎么看" | `03` |
| 预期差检验 | "市场知道吗"、"预期差是什么" | `04` |
| 外围校验 | "外围环境"、"北向资金"、"汇率" | `05` |
| 组合检查 | "仓位怎么配"、"会不会重复" | `06` |
| 持仓审视 | "要不要继续拿"、"季报出了" | `08` |
| 止损决策 | "跌了X%"、"要不要止损" | `09`、`10` |
| 偏误检查 | "我是不是被锚定了"、"情绪化了吗" | `10` |

### 第 2 步：判断使用哪套框架（关键！）

在加载模块前必须先判断目标标的属于哪类：

```
是否满足"高景气成长股"的全部条件？
├── 行业处于新技术/新需求爆发初期？
├── 渗透率 <20%？
├── 增长主要靠"量增"而非"涨价"？
└── 龙头收入增速连续3季度>30%？
   │
   ├── 全部满足 → 用 modules/07-高景气成长股框架.md
   └── 任一不满足 → 用周期拐点策略（modules/01-06, 08-10）
```

**两个框架的退出规则不可混用！** 周期股用 `09-退出纪律.md`，高景气股用 `07` 中的退出纪律。

### 第 3 步：按检查清单逐关评估

加载 `checklist.md` 作为决策骨架，对应到具体模块详细规则：

```
第一关：财报分析     → modules/01
第二关：行业拐点    → modules/02（周期）或 modules/07（高景气）
第三关：估值边际    → modules/03
第四关：预期差确认 → modules/04
第五关：外围校验    → modules/05
第六关：组合检查    → modules/06
建仓后：持仓管理    → modules/08
触发退出：           → modules/09 或 modules/07
全程保障：           → modules/10（认知偏误）
```

**关卡顺序不可跳过：** 任一关 NO 则停止，要么放弃，要么补足条件。

### 第 4 步：用结构化格式输出

回复必须包含以下要素：

1. **场景识别结论**（"这是 XX 场景，使用 YY 框架"）
2. **逐关评估表格**（关卡名 / 标准 / 用户提供的数据 / 是否通过）
3. **关键缺失信息**（需要用户补充哪些数据才能完整判断）
4. **结论与建议**（建仓/审视/退出 + 仓位/动作）
5. **风险提示**（最容易踩的偏误或盲区）

---

## 重要约束

### 必须遵守

- **不能跳过预期差检验（第四关）**：即使前三关都过，无预期差 = 不买
- **不能基于不完整信息给出"建议买入"**：必须列出缺失数据
- **不能混用两套框架的退出规则**：周期股不能用渗透率退出，高景气股不能用毛利率下滑退出
- **必须提示组合层面风险**：单只股看再好，组合层面失控就否决
- **止损触发先减半**：永远不允许"再等等看"占用全部仓位

### 不要做

- 不预测股价走势或给出具体目标价
- 不推荐具体股票（用户提供标的后才评估）
- 不替代用户决策，而是按规则给出结构化判断
- 不省略缺失信息的提示（数据不全就明确说"我无法判断 X 关"）

## 使用示例

### 示例 1：用户问"XX 股票能不能买"

```
用户：帮我看看立讯精密能不能买。Q3 扣非利润+25%，PE 22x 在历史 35% 分位，
苹果链订单回暖，北向最近在流入。

回复（按工作流）：
1. 识别：新建仓评估
2. 框架判断：消费电子，渗透率成熟，增长靠订单/份额而非渗透率提升 → 周期股框架
3. 加载：checklist + modules 01/02/03/04/05/06
4. 逐关评估：
   - 第一关 ✅（部分）：扣非+25%通过，但缺毛利率/现金流数据
   - 第二关 ⚠️：苹果链回暖是单一信号，需配合更多确认
   - 第三关 ✅：PE 35% 分位安全
   - 第四关 ❌：没看到明确的预期差描述
   - 第五关 ✅：北向流入有利
   - 第六关 ?：未提供现有持仓
5. 结论：信息不足以下单，需补充第一/二/四/六关数据
```

### 示例 2：用户问"AI 算力链是不是高景气"

```
用户：现在 CPO 板块涨得很猛，是不是该用高景气框架？

回复：
1. 识别：框架判断
2. 加载 modules/07 检查启用条件：
   - ✅ 新技术爆发初期
   - ✅ 渗透率 <20%（光模块在数据中心的渗透率仍处早期）
   - ✅ 量驱动而非涨价
   - ✅ 龙头收入增速 >30% 多季度
3. 结论：可以用高景气框架
4. 提醒：高景气框架仓位上限 15%，止损线 20%，需对照 modules/07 完整规则
```

### 示例 3：用户问"持仓跌了 18% 要不要走"

```
用户：我持有的 XX 从最高点跌了 18%，但行业月度数据还在改善，
我觉得是市场过度反应想等等。

回复：
1. 识别：止损决策（已触发周期股 15% 止损线）
2. 加载 modules/09 + modules/10
3. 噪音 vs 信号判断：
   - 行业基本面未变 → 倾向噪音
   - 但已触发止损线 → 必须按规则
4. 偏误检查：
   - "想等等看" → 处置效应警告（亏的拿太久）
   - 检查是否在锚定成本价
5. 冷却期规则：先减半（强制），24 小时后再决定剩余
6. 结论：立即卖出 50% 仓位，明天根据情况决定剩余
```

## 终极原则

> **本 Skill 不是给出"买/卖"答案的工具，而是引导用户按规则严格走完决策流程的工具。**
>
> 当信息不足时，明确指出"无法判断 X 关，需要补充 Y 数据"——这比给出一个未经充分验证的"建议"重要得多。
